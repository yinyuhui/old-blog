表布局 ( 非 CSS 表格布局 )

尹玉慧

　　表格布局一般是不太推荐使用的，但它也有可取的地方，比如可以让每一行的所有单元格都有相同的高度，无论单元格中究竟有多少内容，还可以让一列中的单元格有相同的宽度。

一、表显示值 ( display )

table       { display: table; }

tr          { display: table-row; }

thead       { display: table-header-group; }

tbody       { display: table-row-group; }

tfoot       { display: table-footer-group; }

col         { display: table-column; }

colgroup    { display: table-column-group; }

td,th       { display: table-cell; }

caption     { display: table-caption; }

CSS 中列和列组只能接受四种样式 ： border , background , width , visibility .

二、对象插入规则

如果 table-cell 的父元素不是 table-row ， 那么会默认插入一个匿名的 tale-row 元素。当然不仅仅只能插入 table-row ， 还能插入 table 等。

三、表层

CSS 关于表的显示定义的6个层从顶到底分别为： 单元格、 行、 行组、 列、 列组、 表

默认所有元素背景为透明，故底层的背景可以透过上面的元素显示出来

四、表标题 ( caption )

caption-side: top | bottom;     //标题放置在顶部或底部

caption 可以从 table 中继承 color， 但可以有自己的背景，当然 color 也可以直接设置。

caption 与块级元素类似，可以有内边距、边框、背景等，可以使用 text-align 来实现左对齐、 居中、 右对齐

五、表单元格边框 ( border-collapse )

collapse （ 合并单元格边框 ）, separate ( 不合并单元格边框，默认值 )

通常都会将单元格边框合并，这样会美观一些，否则看着会很奇怪？想不到合适的词了....

边框合并的时候样式按照以下优先顺序： double , solid , dashed , dotted , ridge , outside , groove , inset

颜色按照以下优先顺序： cell , row , row group , column , column group , table .

六、边框间隔 ( border-spacing )

虽然边框一般都合并了，根本没啥用，还是写一下，说不定以后用到了呢，至少有东西可以查一下。

border-spacing: 2px;

使用前提： 上面的单元格边框 border-collapse 要为 separate 才能起作用，否则就算设置了也会被忽略掉

七、空单元格 ( empty-cells )

empty-cells: show | hidden

单元格中包含空格也不算是空单元格

八、表大小

宽度设置，要用 

table-layout: fixed | auto ;

指定使用哪种方法确定表宽，而高度都是自动计算的，可以显式设置

如果是固定列宽导致内容放不下，那么 overflow 决定是否剪裁或生成滚动条

九、对齐

text-align | vetical-align : 　　top | bottom | middle | baseline
